<div class="hero gradient-text-animated">
  <h1>What is toon-memory?</h1>
  <p>Persistent memory for AI coding agents</p>
</div>

## The Problem

AI coding agents forget everything between sessions. Every time you start a new session, the agent starts from scratch:

<ul class="staggered-list">
  <li>"Why did we choose X over Y?"</li>
  <li>"What was that bug fix again?"</li>
  <li>"What patterns does this project use?"</li>
</ul>

<div class="comparison-grid">
  <div class="before-box morph-border fade-in-left">
    <h3>Frustrating</h3>
    <p>You have to explain the same context every session</p>
  </div>
  <div class="before-box morph-border fade-in-right">
    <h3>Wasteful</h3>
    <p>Agent spends time re-learning instead of coding</p>
  </div>
</div>

## The Solution

toon-memory provides persistent memory that survives restarts. It stores your decisions, patterns, bugs, and knowledge in a token-efficient format.

<div class="comparison-grid">
  <div class="after-box morph-border fade-in-left">
    <h3>Efficient</h3>
    <p>Agent remembers everything automatically</p>
  </div>
  <div class="after-box morph-border fade-in-right">
    <h3>Fast</h3>
    <p>Instant recall, no re-learning needed</p>
  </div>
</div>

## How It Works

<div class="steps">
  <div class="step fade-in-up">
    <strong>MCP Server</strong> — Runs locally, talks to your agent via stdio
  </div>
  <div class="step fade-in-up">
    <strong>TOON Format</strong> — Stores data in Token-Oriented Object Notation (~40% fewer tokens than JSON)
  </div>
  <div class="step fade-in-up">
    <strong>Per-project memory</strong> — Each project gets `.opencode/memory/data.toon`
  </div>
  <div class="step fade-in-up">
    <strong>Zero config</strong> — Just install and use
  </div>
</div>

## Features

| Feature | Description |
|---------|-------------|
| 8 MCP Tools | Full memory management via Model Context Protocol |
| 7 Agents | OpenCode, VS Code, Claude, Cursor, Windsurf, Cline, Continue |
| TOON Format | 40% fewer tokens than JSON |
| Auto-Archive | Old entries moved to archive automatically |
| Encryption | AES-256-GCM for sensitive data |
| Watch Mode | Auto-backup every N minutes |
